# 视觉维度矩阵 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 `banana-app-prompt-generator` skill 中引入「视觉维度矩阵」，根据应用分类自动分配差异化的色调、渲染风格和构图基调，避免批量生成时多个应用画面风格/色调雷同。

**Architecture:** 在现有 SKILL.md 的「风格速查表」之上新增三层规则：一级分类基础视觉指纹表、三级分类微调表（含主色锁定和禁用色）、批量去重规则。三层规则优先级从高到低：三级微调 > 一级基础 > 原有风格速查表。

**Tech Stack:** Markdown（SKILL.md 纯文本规则），无代码依赖。

```markdown
## 视觉维度矩阵（优先级高于风格速查表）

### 第一层：一级分类基础视觉指纹

根据应用的一级分类，确定基础视觉维度。**三个维度必须同时体现在 prompt 中。**

| 一级分类 | 色调倾向 | 渲染风格 | 构图基调 |
|---------|---------|---------|---------|
| 应用 | 冷色/中性（蓝、灰、白为主） | 扁平插画 / UI感 / 简洁线条 | 极简、留白充足、功能清晰 |
| 游戏 | 高饱和暖色（金、红、深紫为主） | 3D渲染 / 概念艺术 / 史诗插画 | 戏剧性、史诗感、视觉张力强 |
| 小游戏 | 明亮糖果色（黄、绿、橙为主） | 卡通 / Q版 / 像素风 | 活泼、趣味性、轻松欢快 |
| 快应用 | 简洁功能色（品牌主色+白色） | 极简图标感 / 功能示意 | 功能导向、直接、高效 |
```

- [ ] **Step 2: 在第一层表格之后，插入第二层三级分类微调表**

```markdown
### 第二层：三级分类微调（锁定主色 + 禁用色）

在一级基础视觉指纹之上，三级分类进一步锁定主色，并标注禁用色（批量生成时防止撞车的备选方向）。

**已收录三级分类（持续补充中）：**

| 三级分类 | 锁定主色 | 禁用色（批量时避免重复） | 特征关键词 |
|---------|---------|---------|---------|
| 应用-聊天社交-社区 | 珊瑚橙 / 暖粉 | 冷蓝、深灰 | 人与人连接、温暖社交、活跃氛围 |
| 应用-新闻阅读-漫画 | 深蓝 / 墨色 / 暗红 | 糖果色、荧光色 | 内容沉浸、阅读感、叙事张力 |
| 应用-考试学习-语言学习 | 清新绿 / 天蓝 | 深暗色、高饱和红 | 知识成长、轻松学习、进步感 |
| 应用-母婴儿童-儿童启蒙 | 柔黄 / 嫩绿 / 天蓝 | 深色系、高对比色 | 童趣、安全感、亲子温暖 |
| 游戏-经营策略-MOBA竞技 | 战场金 / 热血红 | 粉色、柔和色 | 英雄对决、团战、史诗竞技 |

**未收录三级分类的处理：** 仅使用一级分类的基础视觉指纹，不做额外微调。
```

- [ ] **Step 3: 在第二层之后，插入第三层批量去重规则**

```markdown
### 第三层：批量去重规则（同批次多应用时强制执行）

当用户**一次提供多个应用**需要生成 prompt 时，必须执行以下步骤：

**Step A：生成前，列出「视觉占用表」**

在开始生成任何 prompt 之前，先输出一个简短的占用表：

```
视觉占用表（本批次）：
- [应用名1]：珊瑚橙 × 扁平插画 × 极简
- [应用名2]：深蓝 × 概念艺术 × 戏剧性
...
```

**Step B：检查新应用是否与已有应用撞车**

如果新应用的「色调 + 渲染风格」两个维度与已有应用完全相同，则：
1. 保持色调不变（色调由分类决定，不随意改动）
2. 在渲染风格上做差异化（如：同为扁平插画 → 改为线描插画 / 水彩风 / 剪纸风）
3. 在构图基调上做差异化（如：同为极简 → 改为叙事性构图 / 场景化构图）

**Step C：在 prompt 中明确体现差异化维度**

每个应用的 prompt 必须包含能体现其「视觉指纹」的关键词，确保生成结果可区分。
```
